The Rgifier

  • Category: Research
  • Tools Used: Data Analysis, Python, R
  • Project date: Summer 2023

Project Description

In the summer of 2023, before I went to college, I conducted research at the biochemistry department at James Madison University. This was a joint study with the Lawrence Berkeley National Laboratory in order to develop a software program to automate small angle x-ray scattering (SAXS) data processing.

Across the nation, universities and research labs mail their samples to the SIBYLS beamline at ALS for high throughput SAXS (HT-SAXS) collection. The problem is that HT-SAXS collects 30+ frames of data from a 96 well plate, which produces over 2800 data frames. The common method for processing all this data is through a rather manually driven program known as RAW. For someone well-versed in the software, it takes them around ~2 minutes per well to process data. We wanted to create a software that would automate everything to save time.

Using Python, I developed the Rgifier, which can take in all 96 wells at once, no matter if the samples are proteins, carbohydrates, or DNA, and calculate the radius of gyration (Rg), forward scattering intensity (I0), calculated mass, volume, porod exponent, and suggests the longest series of frames to average, since not all frames are useful data. The Rgifier can compute all this from a raw data set that one would receive back from ALS in approximately 4 seconds. While the analysis is not perfect, in comparison to RAW, it is pretty spot on for proteins. In one summer, there was a lot of potential that came along with this project as showcased in a poster that was presented at Lawrence Berkeley's Advanced Light Source User Meeting.
